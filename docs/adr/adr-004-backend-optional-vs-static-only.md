# ADR-004 — Отказ от backend и переход к полностью статической архитектуре CMS Grav

## Статус
Accepted

## Контекст

Ранее рассматривались варианты:

- портал со статикой;
- портал с REST backend;
- гибрид JSON + HTML.

После финальной архитектурной проработки было решено:

- **не использовать backend вообще**;
- не реализовывать API `/news`, `/status`, `/health`;
- не хранить данные в JSON-файлах;
- не генерировать данные автоматически.

Новости, аварии, ссылки на мастер, сертификаты — **редактируются вручную** в Grav Master.

## Решение

### 1. Портал полностью реализован на CMS Grav

- Grav Master — редактирование
- Grav Slave 1/2 — отдача пользователям
- никаких динамических сервисов
- HTML + Markdown + медиафайлы

### 2. Backend отсутствует

Нет:

- API
- JSON-хранилищ
- микросервисов
- cron-задач
- автоматического обновления статусов

### 3. Вся безопасность обеспечивается отсутствием серверной логики

Статика невозможна к компрометации через код.

## Последствия

### Положительные
- минимальная архитектура;
- нет уязвимостей backend-приложений;
- повышенная отказоустойчивость (2 slave).

### Отрицательные
- администраторы вносят статусы вручную.
