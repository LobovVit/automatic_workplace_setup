# ADR-003 — Полностью локальное выполнение мастера и локальное логирование

## Статус
Accepted

## Контекст

В инфраструктуре ФК важно, чтобы:

- мастер не отправлял ни личные, ни криптографические данные;
- мастер работал даже в условиях отсутствия интернета;
- логирование было доступно локально для отправки в техподдержку;
- проверка конфигурации выполнялась офлайн.

## Решение

### 1. Мастер полностью автономен

- Не делает сетевых запросов.
- Не обращается к серверу ЛК ЭБ.
- Не отправляет отчёты куда-либо самостоятельно.
- Вся работа локальная: CryptoPro, сертификаты, драйверы, реестр, браузер.

### 2. Локальное логирование

Мастер пишет логи:

- в локальный каталог пользователя;
- создаёт файлы типа `master-log-YYYYMMDD.txt`;
- записывает результаты всех контролей (Check+Fix).

### 3. Экспорт отчёта

Пользователь может:

- просмотреть отчёт в UI;
- сохранить его в файл (PDF/TXT/HTML);
- отправить вручную сотруднику поддержки.

---

## Последствия

### Положительные
- полностью безопасно;
- нет зависимости от сети;
- нет риска утечки данных.

### Отрицательные
- нет автоматической отправки логов (по требованиям безопасности).
