# Подсистема автонастройки рабочего места для ЛК ГИИС «Электронный бюджет»

## Состав архитектурного пакета

- `Project.md` — основной архитектурный документ (описание целей, архитектуры, C4, Event Flow, ADR).
- `adr/` — набор Architectural Decision Records (ADR-001…ADR-005).
- `c4/` — PlantUML-диаграммы C4 (уровни 1–4).
- `images/` — PNG-изображения C4-диаграмм и Event Flow.
- `event-flow/` — sequence-диаграмма и текстовый сценарий процесса работы пользователя.

## Используемые технологии

- **CMS Grav** (Master + 2×Slave) — реализация портала и раздача статических файлов.
- **Nginx** — reverse proxy перед Grav.
- **Load Balancer (L4/L7)** — балансировка трафика на Nginx.
- **Avalonia UI + .NET 8 + C# + XAML + MVVM** — мастер автонастройки (Windows / Linux).
- **PlantUML** — C4-диаграммы и Event Flow.
- **Markdown** — основная документация (Project, ADR).

## Кратко об архитектуре

- Портал реализован полностью на CMS Grav, без выделенного backend.
- Grav Master используется для администрирования контента (новости, аварии, инструкции, мастера).
- Grav Slave #1 и #2 отдают HTML/статический контент пользователям через Nginx и Load Balancer.
- Мастер автонастройки — desktop-приложение (Windows / Linux), скачивается с портала и работает локально.
- Все проверки/исправления выполняются локально, без отправки данных наружу.
